計算の5ステップ
STEP 01 · 地点
📍 緯度・経度で
太陽の1年間の動きが決まる
緯度・経度を入力するだけで、365日・24時間すべての太陽の位置（方位角・仰角）が天文学の数式で計算できます。夏は高く、冬は低い軌道を描きます。
必須入力
STEP 02 · 建物情報
🏢 建物の高さ×距離で
影の届く角度がわかる
南側の建物が高いほど・近いほど「遮蔽仰角」が大きくなり、太陽が隠れる時間が増えます。特に冬は太陽が低いため、この影響が最大になります。
精度に最も影響

STEP 03 · 窓の写真（3方向）
📷 写真から遮蔽物マップを
自動生成する
窓の向きに合わせて左・中央・右の3方向を撮影すると、どの方角・どの高さに建物や木があるかを画像解析で検出します。写真中の空の割合（Sky View Factor）が高いほど日照率も高くなります。

STEP 04 · 周辺環境
🌿 写真で検出できない情報を
補足する
落葉樹は季節によって遮蔽率が変わるため、写真だけでは判定が困難です。「南側に落葉樹がある」と入力すると、冬季（11〜3月）に+15%の透過補正が適用されます。
任意・精度向上に有効
STEP 05 · 計算・出力
🎯 年間の日照を
ヒートマップとスコアで表示
12ヶ月×6時間帯（7〜18時）のマトリクスで晴天時の太陽が入る時間帯を算出。窓ごとのスコアを単純平均して物件全体の日照スコアを表示します。

計算式の開示

① 遮蔽仰角の計算

θ_obs = arctan( (建物高さ − 自分の高さ) / 距離 ) // 単位: ラジアン → 度に変換 自分の高さ = 3.0 × (階数 − 1) + 1.5 m // 1階あたり3m、床から目線まで1.5m
南側建物の場合は方位角 135°〜225° の遮蔽マップに反映。窓方向の建物は窓方向 ±60° のアークに反映します。
写真のSVF（Sky View Factor）からも遮蔽仰角を推定します：θ_svf = (1 − SVF) × 30°

② 日照判定（時間スロットごと）

// SAMPLE_DAYS = [1, 8, 15, 22]（月内4日をサンプリング） sunlit = 0 for each sampleDay in [1, 8, 15, 22]: pos = SunCalc.getPosition(date, lat, lng) // → { altitude: rad, azimuth: rad } az_deg = pos.azimuth × (180/π) + 180 // 0=北, 180=南 alt_deg = pos.altitude × (180/π) if alt_deg > 0: // 地平線より上 obstruction = obstMap[round(az_deg)] if alt_deg > obstruction: sunlit += 1 sunlit_ratio = sunlit / 4 // 4サンプルの平均
月ごとに4日分をサンプリングし、各時間スロットの日照ありの日数を集計。
落葉樹補正：11〜3月（冬季）に sunlit_ratio += 0.15（最大1.0にクランプ）

③ 窓スコアと物件全体スコア

// 窓スコア（0〜100） windowScore = mean(heatmap[month][slot] for all months, slots) × 100 × SIZE_MULT[窓サイズ] // SIZE_MULT: 小=0.85, 中=1.0, 大=1.1 // 物件全体スコア（窓の単純平均） totalScore = mean(windowScore for all windows)
窓の向きや数によって物件スコアは変動します。南向きの大きな窓が多いほどスコアは高くなります。

精度と限界

推定誤差の目安

入力情報がそろっている場合の誤差は ±20〜30% が目安です。以下の要因により誤差は大きくなります。
✅ 精度が高い条件
・緯度・経度が正確に入力されている
・南側建物の高さ・距離を計測済み
・窓ごとに3方向の写真をアップロード
・写真が晴天時・空が広く映っている
・高層階（遮蔽物が少ない）
⚠️ 誤差が大きくなる条件
・建物情報を「わからない」で入力
・写真なし（推定のみで計算）
・複数の建物・不規則な地形
・大きな落葉樹が隣接している
・写真が曇天・逆光・室内から撮影
❌ 考慮できないもの
・窓ガラスの透過率・コーティング
・局所的な霧・雲・PM2.5
・落葉樹の葉の量（季節内変動）
・反射光・地面からの散乱光
・隣接建物の将来的な変化
🏗 建築前リサーチご利用の場合
  • 更地の中央から撮影するため、実際の窓位置（角部屋・南寄り等）によって誤差が生じます
  • 撮影者の身長・カメラ位置による誤差は計算上無視しています
  • 階数が高くなるほど周辺建物の影響は小さくなりますが、完全な遮蔽計算ではありません
  • 本ツールは土地選定の参考値です。正確なシミュレーションは建築士にご相談ください

用途によって何が変わるのか
🏠 居住・不動産 ›
計算：晴天理論値 重視：冬 出力：月別×時間帯ヒートマップ

「晴れた日にどれだけ日が入るか」が居住判断の基準になるため、気象補正なしの晴天理論値を使用。冬の日照が最も生活快適性に影響するため、冬を重視した評価を行います。電気代・家具の日焼け・リビングの明るさへの影響をヒートマップで可視化します。

🏗 建築前リサーチ ›
計算：晴天理論値 重視：冬 出力：ヒートマップ＋階数シミュレーター

更地や設計前の段階で、階数ごとの日照率を比較できます。周囲の建物による遮蔽角度を計算し、「日照率60%以上には○階以上が必要」という設計判断をサポートします。

追加機能：階数シミュレーター
1階〜予定最上階までの日照率を一覧表示。最適な階数の目安を設計段階から把握できます。
🌱 農業・園芸 ›
計算：気象補正あり 重視：春〜秋（生育期間） 出力：月別積算日射量グラフ

実際の栽培環境では雨・曇りの日も影響するため、気象補正を適用した実効日射量を使用。春〜秋の生育期間に注目し、作物・植物の種類ごとの最低日照要件と照合して適性を評価します。

☀️ ソーラー・エネルギー ›
計算：気象補正あり（年間積算） 重視：年間トータル 出力：積算日射量＋最適設置角度

発電量は年間の実際の日射量で決まるため、気象補正を適用した年間積算値を使用。最適な設置角度（傾斜角・方位角）の提案と、発電効率の目安を算出します。

🏪 商業・施設 ›
計算：晴天理論値 重視：営業時間帯 出力：時間帯別ヒートマップ

店舗・カフェのテラス席や保育園の園庭など、営業・利用時間帯に絞った日照評価を行います。「ランチタイムに日が当たるか」「夕方の西日が強すぎないか」など、商業利用に特有の観点で分析します。

⚠️ 免責事項：この計算は天文学データと入力情報をもとにした推定値です。樹木の季節変動・局所的な天候・地形の微細な差異により実際の日照は異なります。誤差の目安は ±20〜30% です。不動産・建築の意思決定には専門家の調査を併用してください。
